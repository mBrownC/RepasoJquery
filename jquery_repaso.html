Tema 1

¿Qué es AJAX?

AJAX = Asynchronous JavaScript And XML

Técnica de desarrollo web para crear aplicaciones interactivas que se comunican con el servidor sin recargar la página completa.

Concepto Principal

AJAX permite que las páginas web se actualicen de forma asincrónica, intercambiando datos con el servidor en segundo plano.

¿Por qué es importante?

  • Experiencia de usuario mejorada: Las páginas responden más rápido
  • Ahorro de ancho de banda: Solo se transfieren los datos necesarios
  • Interactividad: Actualizaciones en tiempo real sin recargar
  • Aplicaciones modernas: Gmail, Facebook, Twitter usan AJAX constantemente

Ejemplo del mundo real

🔍 Buscador de Google: Cuando escribes en el buscador, ves sugerencias automáticas. Esas sugerencias se cargan usando AJAX, sin recargar toda la página.

❌ Sin AJAX (Antiguo)

1. Usuario hace click en botón

2. Página completa se recarga

3. Usuario espera...

4. Nueva página se muestra

⏱️ Lento y molesto

✅ Con AJAX (Moderno)

1. Usuario hace click en botón

2. Petición al servidor (invisible)

3. Datos regresan

4. Solo se actualiza una parte

⚡ Rápido y fluido

¿Cómo funciona AJAX?

// Ejemplo básico de AJAX con JavaScript puro
fetch('https://api.ejemplo.com/datos')
    .then(response => response.json())
    .then(datos => {
        // Actualizar solo una parte de la página
        document.getElementById('resultado').innerHTML = datos;
    });

AJAX con jQuery

jQuery simplifica enormemente el uso de AJAX. Por eso se menciona tanto en el contexto de aplicaciones web modernas.

// AJAX con jQuery (mucho más simple)
$.ajax({
    url: 'https://api.ejemplo.com/usuarios',
    method: 'GET',
    success: function(datos) {
        // Hacer algo con los datos
        console.log(datos);
    }
});
💡 Importante: Aunque AJAX tiene "XML" en su nombre, hoy en día se usa principalmente JSON como formato de datos por ser más ligero y fácil de usar.

Comparación: JavaScript vs jQuery

JavaScript Puro

// Seleccionar elementos
document.getElementById('miDiv');
document.querySelectorAll('.miClase');

// Cambiar contenido
elem.innerHTML = 'Texto';

// Agregar evento
elem.addEventListener('click', fn);

jQuery

// Seleccionar elementos
$('#miDiv');
$('.miClase');

// Cambiar contenido
$('#miDiv').html('Texto');

// Agregar evento
$('#miDiv').on('click', fn);

El símbolo $ (dólar)

$ es simplemente un alias de jQuery. Son exactamente lo mismo:

$('p') = jQuery('p')

¿Cuándo usar jQuery?

✅ Usar jQuery ❌ No usar jQuery
Proyectos legacy existentes Aplicaciones React/Vue/Angular nuevas
Soporte a navegadores antiguos (IE) Solo navegadores modernos (ES6+)
Prototipado rápido Proyectos que requieren máximo rendimiento
Uso de plugins jQuery específicos Cuando quieres minimizar dependencias
⚠️ Estado actual (2025): jQuery sigue siendo usado en millones de sitios web, pero para proyectos nuevos se recomienda JavaScript moderno (ES6+) o frameworks como React, Vue o Angular.

Selectores jQuery

$(selector) - El corazón de jQuery

Los selectores permiten "encontrar" elementos en el HTML para manipularlos.

Selectores Básicos

Selector Ejemplo Selecciona
$('p') Por etiqueta Todos los elementos <p>
$('#miId') Por ID El elemento con id="miId"
$('.miClase') Por clase Todos los elementos con class="miClase"
$('*') Universal TODOS los elementos

Selectores Combinados

// Descendiente: todos los <li> dentro de <ul>
$('ul li')

// Hijo directo: solo los <li> hijos directos de <ul>
$('ul > li')

// Múltiples selectores: todos los <p> y todos los <div>
$('p, div')

// Con clase específica: <p> con class="destacado"
$('p.destacado')

Selectores por Atributo

// Elementos con atributo "name"
$('[name]')

// Input con type="text"
$('input[type="text"]')

// Links que empiezan con "https"
$('a[href^="https"]')

// Imágenes que terminan en ".png"
$('img[src$=".png"]')

Selectores de Filtro

Posición

// Primer <li>
$('li:first')

// Último <li>
$('li:last')

// Pares (2, 4, 6...)
$('li:even')

// Impares (1, 3, 5...)
$('li:odd')

Estado

// Ocultos
$(':hidden')

// Visibles
$(':visible')

// Checkboxes marcados
$(':checked')

// Inputs deshabilitados
$(':disabled')

Ejemplo Práctico

// HTML
<ul id="miLista">
    <li class="activo">Item 1</li>
    <li>Item 2</li>
    <li class="activo">Item 3</li>
</ul>

// jQuery
$('#miLista')              // Selecciona el <ul>
$('#miLista li')           // Todos los <li> (3 elementos)
$('#miLista li.activo')   // Solo los <li> con class="activo" (2 elementos)
$('#miLista li:first')    // Primer <li>
Tema 3

Manipulación del DOM con jQuery

Métodos para modificar contenido

.text() - Texto plano

// OBTENER texto
var textoActual = $('p').text();

// ESTABLECER texto (sin HTML)
$('p').text('Nuevo texto');

.html() - Contenido HTML

// OBTENER HTML
var htmlActual = $('div').html();

// ESTABLECER HTML (con etiquetas)
$('div').html('<strong>Texto en negrita</strong>');

.val() - Valores de inputs

// OBTENER valor de input
var valor = $('input').val();

// ESTABLECER valor
$('input').val('Nuevo valor');

// Limpiar input
$('input').val('');
⚠️ Diferencia importante:
  • .text() - Seguro, no ejecuta HTML/scripts
  • .html() - Puede ejecutar scripts, ¡cuidado con datos de usuarios!

Métodos para modificar estilos

// Cambiar UNA propiedad CSS
$('p').css('color', 'red');

// Cambiar MÚLTIPLES propiedades
$('p').css({
    'color': 'red',
    'font-size': '20px',
    'background-color': 'yellow'
});

// OBTENER valor de propiedad CSS
var colorActual = $('p').css('color');

Manipular clases CSS

// Agregar clase
$('p').addClass('activo');

// Quitar clase
$('p').removeClass('activo');

// Alternar clase
$('p').toggleClass('activo');
// Verificar si tiene clase
if ($('p').hasClass('activo')) {
    console.log('Tiene la clase');
}

Manipular atributos

// OBTENER atributo
var href = $('a').attr('href');

// ESTABLECER atributo
$('a').attr('href', 'https://google.com');

// ESTABLECER múltiples atributos
$('img').attr({
    'src': 'imagen.png',
    'alt': 'Descripción'
});

// ELIMINAR atributo
$('a').removeAttr('target');
Tema 3

Crear y Agregar Elementos al DOM

Crear elementos nuevos

// Crear elemento vacío
var nuevoDiv = $('<div></div>');

// Crear elemento con contenido
var nuevoP = $('<p>Este es un párrafo nuevo</p>');

// Crear y configurar en cadena (method chaining)
var nuevoLi = $('<li></li>')
    .text('Item nuevo')
    .addClass('activo')
    .css('color', 'red');

Métodos para insertar elementos

Método Descripción Ejemplo
.append() Agrega al FINAL (adentro) $('#lista').append(nuevoLi)
.prepend() Agrega al INICIO (adentro) $('#lista').prepend(nuevoLi)
.after() Agrega DESPUÉS (afuera) $('p').after(nuevoDiv)
.before() Agrega ANTES (afuera) $('p').before(nuevoDiv)
.appendTo() Igual append, orden invertido nuevoLi.appendTo('#lista')
.prependTo() Igual prepend, orden invertido nuevoLi.prependTo('#lista')

Diferencia entre append() y appendTo()

.append() - El padre recibe

// Sintaxis: padre.append(hijo)
$('#lista').append('<li>Nuevo</li>');

.appendTo() - El hijo va hacia

// Sintaxis: hijo.appendTo(padre)
$('<li>Nuevo</li>').appendTo('#lista');

Ejemplo visual: append vs prepend vs after vs before

// HTML inicial
<div id="contenedor">
    <p>Párrafo existente</p>
</div>

// jQuery
$('#contenedor').append('<p>Al final (dentro)</p>');
$('#contenedor').prepend('<p>Al inicio (dentro)</p>');
$('#contenedor').after('<p>Después (fuera)</p>');
$('#contenedor').before('<p>Antes (fuera)</p>');

// Resultado:
<p>Antes (fuera)</p>
<div id="contenedor">
    <p>Al inicio (dentro)</p>
    <p>Párrafo existente</p>
    <p>Al final (dentro)</p>
</div>
<p>Después (fuera)</p>

Eliminar elementos

// Eliminar elemento (y sus hijos)
$('#miDiv').remove();

// Vaciar contenido (solo elimina hijos, no el elemento)
$('#miDiv').empty();
💡 Method Chaining: jQuery permite encadenar métodos porque cada método devuelve el objeto jQuery:
$('p').addClass('activo').css('color', 'red').text('Hola');
Tema 4

Eventos en jQuery

¿Qué son los eventos?

Los eventos son acciones que ocurren en la página web: clicks, movimientos del mouse, teclas presionadas, etc.

Tipos de eventos más comunes

🖱️ Mouse

  • click
  • dblclick
  • mouseenter
  • mouseleave
  • mouseover
  • mouseout

⌨️ Teclado

  • keydown
  • keyup
  • keypress

📝 Formularios

  • focus
  • blur
  • change
  • submit

Método .on() - La forma moderna

// Sintaxis básica
$('selector').on('evento', function() {
    // Código a ejecutar cuando ocurre el evento
});

// Ejemplo: Click en un botón
$('#miBoton').on('click', function() {
    alert('¡Botón clickeado!');
});

Atajos de eventos (métodos directos)

// Estas dos formas son equivalentes:

// Forma 1: Con .on()
$('p').on('click', function() {
    console.log('Click!');
});

// Forma 2: Atajo directo .click()
$('p').click(function() {
    console.log('Click!');
});

$(this) - Elemento que disparó el evento

// Cambiar solo el elemento clickeado
$('p').on('click', function() {
    // $(this) se refiere al <p> específico que fue clickeado
    $(this).css('color', 'red');
    $(this).addClass('seleccionado');
});

El objeto event (eventObject)

// Acceder a información del evento
$('input').on('click', function(event) {
    console.log('Tipo de evento:', event.type);         // "click"
    console.log('Elemento:', event.target);             // El elemento clickeado
    console.log('Coordenadas X:', event.pageX);        // Posición X del click
    console.log('Coordenadas Y:', event.pageY);        // Posición Y del click
    
    // Prevenir comportamiento por defecto
    event.preventDefault();
});
💡 preventDefault(): Previene la acción por defecto del navegador. Ejemplo: evitar que un formulario se envíe, o que un link navegue.
Tema 4

Eventos jQuery - Técnicas Avanzadas

Vincular múltiples eventos

Opción 1: Múltiples eventos, misma función

// Ejecutar la misma función en varios eventos
$('input').on('focus blur', function(e) {
    console.log('Evento:', e.type); // Mostrará "focus" o "blur"
});

Opción 2: Objeto con eventos y funciones diferentes

// Cada evento tiene su propia función
$('p').on({
    'click': function() {
        console.log('Clickeado');
    },
    'mouseenter': function() {
        $(this).css('background-color', 'yellow');
    },
    'mouseleave': function() {
        $(this).css('background-color', 'white');
    }
});

Pasar datos al evento

// Sintaxis: .on(eventos, data, handler)
$('button').on('click', { usuario: 'Miguel', rol: 'Instructor' }, function(e) {
    console.log('Usuario:', e.data.usuario); // "Miguel"
    console.log('Rol:', e.data.rol);         // "Instructor"
});

.one() - Evento de una sola vez

// Se ejecuta SOLO LA PRIMERA VEZ
$('p').one('click', function() {
    console.log('Primera vez clickeado');
    // Después de este click, el evento se desvincula automáticamente
});

// Caso de uso: Mensaje de bienvenida que aparece solo una vez
$('#btnComenzar').one('click', function() {
    alert('¡Bienvenido! Este mensaje solo aparece una vez.');
});

.off() - Desvincular eventos

// Desvincular TODOS los eventos de un elemento
$('p').off();

// Desvincular evento específico
$('p').off('click');

// Desvincular función específica (debe ser nombrada)
function miHandler() {
    console.log('Click!');
}

$('p').on('click', miHandler);     // Vincular
$('p').off('click', miHandler);    // Desvincular solo esta función

Event Namespaces (Espacios de nombres)

Permiten organizar y desvincular eventos de forma más granular:

// Vincular con namespace
$('p').on('click.miNamespace', function() {
    console.log('Click con namespace');
});

$('p').on('mouseenter.miNamespace', function() {
    console.log('MouseEnter con namespace');
});

// Desvincular SOLO eventos con ese namespace
$('p').off('.miNamespace'); // Desvincula AMBOS eventos

// Desvincular evento específico del namespace
$('p').off('click.miNamespace'); // Solo el click
💡 Ventajas de los namespaces:
  • Organizar eventos de diferentes módulos o plugins
  • Desvincular grupos de eventos sin afectar otros
  • Evitar conflictos entre diferentes partes del código

$(document).ready() - Esperar que cargue el DOM

// Forma completa
$(document).ready(function() {
    // Código que se ejecuta cuando el DOM está listo
    $('p').click(function() {
        alert('Click!');
    });
});

// Forma abreviada (más común)
$(function() {
    // Todo tu código jQuery aquí
});
Tema 4

Delegación de Eventos

El problema con elementos dinámicos

❌ Esto NO funciona para elementos creados después:

// Vincular eventos a elementos existentes
$('.boton').on('click', function() {
    alert('Click!');
});

// Crear nuevo botón DESPUÉS
$('body').append('<button class="boton">Nuevo</button>');
// ❌ Este botón NO responderá al click

Solución: Delegación de eventos

Vincular el evento a un elemento padre que siempre existe, y especificar el selector hijo:

✅ Esto SÍ funciona para elementos dinámicos:

// Sintaxis: $(padreEstatico).on('evento', 'selector hijo', function)
$('body').on('click', '.boton', function() {
    alert('Click!');
});

// Ahora CUALQUIER botón con class="boton" funcionará
// Incluso los que se creen después
$('body').append('<button class="boton">Nuevo</button>');
// ✅ Este botón SÍ responderá al click

¿Cómo funciona?

  1. El evento se vincula al body (elemento padre estático)
  2. Cuando ocurre un click en cualquier parte del body
  3. jQuery verifica si el elemento clickeado coincide con .boton
  4. Si coincide, ejecuta la función

Ejemplo práctico: Lista dinámica

<ul id="lista">
    <li>Item 1 <button class="eliminar">X</button></li>
    <li>Item 2 <button class="eliminar">X</button></li>
</ul>
<button id="agregar">Agregar Item</button>

<script>
// Delegación: funciona para botones nuevos también
$('#lista').on('click', '.eliminar', function() {
    $(this).parent().remove(); // Elimina el <li> padre
});

// Agregar nuevos items dinámicamente
var contador = 3;
$('#agregar').on('click', function() {
    var nuevoItem = '<li>Item ' + contador + ' <button class="eliminar">X</button></li>';
    $('#lista').append(nuevoItem);
    contador++;
});
</script>

Comparación: Directo vs Delegado

Aspecto Evento Directo Evento Delegado
Sintaxis $('.btn').on('click', fn) $('body').on('click', '.btn', fn)
Elementos dinámicos ❌ No funciona ✅ Sí funciona
Rendimiento Más rápido (menos verificaciones) Ligeramente más lento
Memoria Más uso (1 listener por elemento) Menos uso (1 listener total)
Cuándo usar Elementos estáticos Elementos dinámicos
💡 Regla práctica:
  • Si los elementos ya existen en el HTML → evento directo
  • Si los elementos se crean con JavaScript → delegación
Práctica 1

Quiz Interactivo - Pregunta 1

Pregunta: ¿Qué hace este código?

$('p').on('click', function() {
    $(this).css('color', 'red');
});

Elige la respuesta correcta:

A) ❌

Cambia el color de TODOS los párrafos a rojo inmediatamente

B) ✅

Cambia el color a rojo solo del párrafo que se clickea

C) ❌

Cambia el color del primer párrafo solamente

✅ Respuesta correcta: B

Explicación:

  • $('p') selecciona TODOS los párrafos
  • .on('click', function()) vincula el evento a cada uno
  • $(this) se refiere solo al párrafo específico que fue clickeado
  • Por lo tanto, solo ese párrafo cambiará de color

Práctica 3

Quiz Interactivo - Pregunta 3

Pregunta 3: ¿Cuál es la diferencia?

Código A:

$('#div1').text('<strong>Hola</strong>');

Código B:

$('#div2').html('<strong>Hola</strong>');

¿Qué se mostrará en pantalla?

A) ✅

div1: <strong>Hola</strong>
div2: Hola (en negrita)

B) ❌

Ambos mostrarán Hola en negrita

C) ❌

Ambos mostrarán <strong>Hola</strong>

✅ Respuesta correcta: A

Explicación:

  • .text() inserta el contenido como texto plano, sin interpretar HTML
  • Por eso div1 mostrará literalmente: <strong>Hola</strong>
  • .html() inserta el contenido como HTML, interpretando las etiquetas
  • Por eso div2 mostrará: Hola (la palabra en negrita)
  • Seguridad: Usa .text() con datos de usuarios para evitar XSS
Práctica 4

Quiz Interactivo - Pregunta 4

Pregunta 4: ¿Dónde se agregará el elemento?

// HTML inicial:
<ul id="lista">
    <li>Item 1</li>
    <li>Item 2</li>
</ul>

// jQuery:
$('#lista').prepend('<li>Item Nuevo</li>');

¿Cuál será el resultado final?

A) ✅

• Item Nuevo
• Item 1
• Item 2

B) ❌

• Item 1
• Item 2
• Item Nuevo

C) ❌

• Item 1
• Item Nuevo
• Item 2

✅ Respuesta correcta: A

Explicación:

  • .prepend() agrega el elemento al INICIO del contenedor (como primer hijo)
  • .append() agrega el elemento al FINAL del contenedor (como último hijo)
  • En este caso, "Item Nuevo" se inserta antes de "Item 1"
💡 Recordatorio:
• .prepend() = Al inicio (dentro)
• .append() = Al final (dentro)
• .before() = Antes (fuera)
• .after() = Después (fuera)

Práctica 2

Solución: Lista de Tareas

Código JavaScript completo

$(document).ready(function() {
    
    // 1. Agregar nueva tarea
    $('#btnAgregar').on('click', function() {
        // Obtener el valor del input
        var textoTarea = $('#nuevaTarea').val();
        
        // Validar que no esté vacío
        if (textoTarea.trim() === '') {
            alert('Por favor escribe una tarea');
            return;
        }
        
        // Crear nuevo elemento <li> con botón eliminar
        var nuevaTarea = '<li>' + textoTarea + ' <span class="eliminar">X</span></li>';
        
        // Agregar a la lista
        $('#listaTareas').append(nuevaTarea);
        
        // Limpiar el input
        $('#nuevaTarea').val('');
    });
    
    // 2. Tachar tarea al hacer click (delegación)
    $('#listaTareas').on('click', 'li', function() {
        $(this).toggleClass('completada');
    });
    
    // 3. Eliminar tarea (delegación)
    $('#listaTareas').on('click', '.eliminar', function(e) {
        // Prevenir que el click llegue al <li> (evita tachar al eliminar)
        e.stopPropagation();
        
        // Eliminar el <li> padre
        $(this).parent().remove();
    });
    
    // BONUS: Agregar tarea con Enter
    $('#nuevaTarea').on('keypress', function(e) {
        if (e.which === 13) { // Tecla Enter
            $('#btnAgregar').click();
        }
    });
    
});

Resumen - AJAX y jQuery

Lo que aprendimos hoy

🌐 AJAX

  • Técnica para actualizar páginas sin recargar
  • Usado en todas las aplicaciones modernas
  • JSON como formato de datos preferido
  • jQuery simplifica enormemente AJAX

📚 jQuery Básico

  • Biblioteca que simplifica JavaScript
  • Sintaxis: $(selector).metodo()
  • Instalación vía CDN
  • Cross-browser compatible

🎯 Selectores

  • $('#id') - Por ID
  • $('.clase') - Por clase
  • $('etiqueta') - Por etiqueta
  • Selectores combinados y filtros

✏️ Manipulación DOM

  • .text() - Texto plano
  • .html() - Contenido HTML
  • .val() - Valores de inputs
  • .css() - Estilos CSS
  • .addClass/removeClass()
  • .attr() - Atributos

➕ Crear/Agregar Elementos

  • $('<elemento>') - Crear
  • .append() - Al final (dentro)
  • .prepend() - Al inicio (dentro)
  • .after() - Después (fuera)
  • .before() - Antes (fuera)

⚡ Eventos

  • .on(evento, handler)
  • $(this) - Elemento actual
  • .one() - Una sola vez
  • .off() - Desvincular
  • Delegación para elementos dinámicos
  • Namespaces para organización
